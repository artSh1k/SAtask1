# Глоссарий

| Термин | Определение |
|:---|:---|
| **Туроператор** | Компания, формирующая туристический продукт (тур) или отдельные услуги. В контексте системы — пользователь-менеджер, работающий с бронированиями |
| **Клиент (Турист)** | Конечный потребитель услуги, для которого производится бронирование. Не является прямым пользователем B2B-системы, но его данные — центр бизнес-логики |
| **Поставщик (Отель)** | Юридическое лицо или ИП, предоставляющее услуги размещения и передающее данные о доступности номеров |
| **Квота (Стоп-сейл)** | Выделенный поставщиком пул номеров/мест, гарантированный для туроператора, с возможностью отслеживания остатков в реальном времени |
| **Неттинг** | Процесс получения финальных подтверждённых цен от поставщика (часто ниже заявленных изначально), применяется для расчёта конечной маржи туроператора |
| **Ваучер** | Документ, выдаваемый на руки клиенту, подтверждающий его право на заселение в отель. Может быть документом строгой отчётности или свободной формы |
| **Тайм-лимит** | Срок, в течение которого туроператор должен оплатить бронь, иначе отель вправе автоматически её аннулировать |
| **Suppliers API / GDS** | Глобальные системы дистрибуции или прямые API отелей (CRS/Channel Managers), через которые система туроператора получает данные о ценах и доступности номеров онлайн |
| **Холдирование (Hold)** | Временная блокировка номера на период, пока менеджер связывается с клиентом для подтверждения покупки |

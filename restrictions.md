# Ограничения и бизнес-правила

## Технологические ограничения

- Взаимодействие с Поставщиками — только по протоколам HTTPS/JSON или SOAP/XML (зависит от провайдера)
- Система не может редактировать данные, полученные из API Поставщика. Цена и доступность — read-only. Система может только применить свою наценку поверх входящей цены
- Максимальное время ожидания ответа от внешнего API — 30 секунд. При тайм-ауте система показывает сообщение «Поставщик временно недоступен» и предлагает повторить поиск

## Бизнес-правила

| № | Правило | Логика |
|:---|:---|:---|
| BR-01 | Автоотмена зависших броней | Если статус «В обработке» сохраняется более 24 часов, система автоматически отправляет запрос на отмену Поставщику и переводит бронь в статус «Отменено» |
| BR-02 | Ограничение на отмену со штрафами | Отмена брони, по которой уже наступил штраф 100%, невозможна без подтверждения роли «Старший менеджер» (второй фактор подтверждения) |
| BR-03 | Валидация дат | Запрещено создание брони на прошедшую дату. Дата выезда не может быть раньше даты заезда. Минимальный период бронирования — 1 ночь |
| BR-04 | Офлайн-брони | Брони, созданные по телефону (без мгновенного API-подтверждения), должны вноситься в систему вручную, но маркироваться признаком «Офлайн-заявка» для корректной отчётности |

## Ограничения данных (валидация)

| Поле | Ограничение |
|:---|:---|
| Имя и фамилия туриста | Только буквы (кириллица или латиница, в зависимости от направления), дефис и пробел. Обязательно для заполнения |
| Дата заезда | Не ранее текущей даты. Не позднее 365 дней от текущей даты |
| Количество гостей | Целое число от 1 до 10 |
| Возраст детей | При наличии детей — точный возраст на дату заезда (для корректного подбора тарифа и доп. мест) |

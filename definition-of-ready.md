# Критерии готовности (Definition of Ready)

| № | Критерий | Как проверить |
|:---|:---|:---|
| 1 | Утверждена модель данных | Готова ER-диаграмма сущностей: «Бронь», «Расселение» (Rooms), «Турист», «Поставщик», «Тариф». Определены все атрибуты, типы данных, связи |
| 2 | Определены все состояния объекта | Построена и согласована диаграмма переходов статусов брони (Новая → В обработке → Подтверждена → Оплачена / Аннулирована / Отклонена). Для каждого перехода описан триггер |
| 3 | Согласованы макеты интерфейсов | Утверждены UI/UX-макеты для экранов: «Поисковая выдача», «Карточка брони», «Форма генерации ваучера». Проработаны граничные состояния: пустой поиск, ошибка загрузки, «нет данных» |
| 4 | Описаны API-контракты | Спецификации обмена с внешними Поставщиками (минимум 2-х для пилота) описаны в формате Swagger/OpenAPI. Указаны обязательные и опциональные поля, коды ошибок |
| 5 | Детализированы User Stories | Каждая US разбита по CRUD/LSHAX: описано, кто имеет право, при каких условиях, какие уведомления уходят, что попадает в историю изменений и архив |
| 6 | Расчётные формулы зафиксированы | Формула «Итоговая цена = Цена поставщика (нетто) + Наценка туроператора (%) + Доп. услуги» утверждена фин. отделом |

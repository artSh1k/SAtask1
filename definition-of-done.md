# Критерии завершённости (Definition of Done)

| № | Критерий | Детали |
|:---|:---|:---|
| 1 | Позитивный сценарий пройден | Пользователь успешно создал бронь, получил подтверждение по API, внёс оплату и сгенерировал PDF-ваучер. Весь путь проходится без ошибок |
| 2 | Негативные сценарии отработаны | При отказе Поставщика бронь переходит в статус «Отклонена» с сохранением причины отказа. При тайм-ауте API система показывает понятное сообщение и не «зависает» |
| 3 | Расчёты корректны | Итоговая цена для клиента везде соответствует формуле: Цена нетто + Наценка Брутто. Проверено на 5+ тестовых примерах с разными ставками наценки |
| 4 | Интеграционное тестирование пройдено | Пройдены юнит-, интеграционные и E2E-тесты. В том числе — имитация отказа стороннего API, тайм-аута, получения неполных данных от провайдера |
| 5 | Аудит действий включён | Все смены статуса брони логируются в историю: кто изменил, когда (timestamp), старое значение, новое значение, IP-адрес. История доступна в карточке брони |
| 6 | Фоновые процессы работают | Настроен и протестирован Job, автоматически отменяющий брони с истекшим тайм-лимитом (если не внесена оплата). Время срабатывания — каждые 15 минут |
| 7 | Документация для пользователей готова | Разработана и выложена краткая инструкция для Менеджера по работе с блоком «Бронирование отелей» (формат: PDF или страница в Confluence/Notion) |
